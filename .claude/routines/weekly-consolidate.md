# Weekly Consolidate (Friday rollup)

You run **once a week, Friday afternoon**. For each in-scope repo that has commits on the weekly agent branch, open **one consolidation PR** against `main` for the user's manual review. You do **not** auto-merge — this PR is the user's single weekly review touchpoint.

You are running unattended. Be safe — don't merge anything, don't delete branches, don't force-push.

---

## Tooling and workspace

**Workspace.** Repo paths below are written as `~/git/<repo>` (local convention). In a remote Claude Code runtime where the source repo is auto-cloned to a different path, treat each `~/git/<repo>` reference as "the local clone of that repo in your environment." Clone any missing siblings with `git clone https://github.com/Rezivure/<repo>.git` next to the existing clone.

**GitHub operations.** This prompt uses `gh` CLI commands as canonical operation names. If `gh` is not available in your runtime, use the equivalent **GitHub MCP tools** for the same operations (list/get/create/update PRs, comment, label). What matters is the operation, not the command syntax. For `git` operations (fetch, log, rev-list), use the `git` CLI directly.

## Phase 0 — Setup

1. Compute:
   - `TODAY=$(date +%Y-%m-%d)`
   - `MONDAY=$(date -v-Mon +%Y-%m-%d)` (Monday of *this* week — the one ending today)
   - `WEEKLY_BRANCH="agent/week-of-${MONDAY}"`

2. For each of the 5 repos:
   - If local clone is missing: `gh repo clone Rezivure/<name> ~/git/<name>`
   - `cd ~/git/<name>`
   - `git fetch origin --prune`

## Phase 1 — Per-repo consolidation

For each of the 5 repos:

### Step A — Skip if no work

- If `origin/$WEEKLY_BRANCH` doesn't exist: skip this repo.
- Compute commits ahead of main:
  ```
  ahead=$(git rev-list --count origin/main..origin/$WEEKLY_BRANCH 2>/dev/null || echo 0)
  ```
- If `ahead == 0`: skip.

### Step B — Gather what landed this week

Collect:

1. **Merged PRs** that landed on the weekly branch:
   ```
   gh pr list --repo Rezivure/<repo> --base "$WEEKLY_BRANCH" --state merged --limit 100 \
     --json number,title,url,body,mergedAt,headRefName
   ```
   For each, extract from the PR body which Grid-Mobile issue it addressed (look for `Addresses Rezivure/Grid-Mobile#<n>` or `Fixes #<n>`).

2. **Open stuck PRs** against the weekly branch — escalations:
   ```
   gh pr list --repo Rezivure/<repo> --base "$WEEKLY_BRANCH" --state open --label agent-stuck \
     --json number,title,url,body,labels
   ```

### Step C — Build PR body

```markdown
## Summary — week of <MONDAY> (<repo>)

Merged this week (<count>):
- Rezivure/<repo>#<pr> — <pr title> _(addresses Grid-Mobile#<issue>)_
- ...

## Stuck — needs you (<count>)

These were attempted but failed review/CI 3 times. Pull them open, look at the latest reviewer comment, and either fix manually or close.

- Rezivure/<repo>#<pr> — <pr title> _(addresses Grid-Mobile#<issue>)_
  Latest reviewer reason: <one line>
- ...

(If no stuck items, omit this section entirely.)

## Test plan

- [ ] Skim each merged PR's diff via the "Files changed" tab on this rollup PR.
- [ ] CI already ran per individual PR; this rollup re-runs CI on the combined diff.
- [ ] Merge this PR to ship the week into `main`.

_Generated by `.claude/routines/weekly-consolidate.md` on <TODAY>. Agent-managed branch: `<WEEKLY_BRANCH>`._
```

### Step D — Open or update the consolidation PR

Check if a consolidation PR already exists for this week (e.g., if Friday cron re-ran or you're updating mid-week):

```
existing=$(gh pr list --repo Rezivure/<repo> --base main --head "$WEEKLY_BRANCH" --state open --json number --jq '.[0].number')
```

- If `existing` is empty:
  ```
  gh pr create \
    --repo Rezivure/<repo> \
    --base main \
    --head "$WEEKLY_BRANCH" \
    --title "Weekly agent rollup: week of $MONDAY" \
    --body "<body from Step C>"
  ```
- Else: update the body in place.
  ```
  gh pr edit "$existing" --repo Rezivure/<repo> --body "<body from Step C>"
  ```

**Do NOT merge.** This PR exists for the user to review and merge manually.

## Guardrails

- **Never merge the weekly rollup PR** — the user does that manually.
- **Never delete the weekly branch** — it stays alive until either merged or naturally aging out (next Monday a new weekly branch starts; the old one persists until you merge or close it).
- **Never force-push, never `--no-verify`, never amend a published commit.**
- If `gh pr create` fails because there are no commits ahead of main (already-merged scenario): skip silently, not an error.
- If a previous week's rollup PR is still open against `main`, **do not touch it** — that's the user's outstanding review queue, not this week's work.
